# E2E Tests

End-to-end tests for the Mentor AI application using [Playwright](https://playwright.dev/) and the `@iblai/iblai-js/playwright` SDK.

## Prerequisites

- Node.js 18+
- pnpm
- Playwright browsers installed

```bash
pnpm install
npx playwright install
```

## Environment Setup

Copy the example env file and fill in your credentials:

```bash
cp e2e/.env.example e2e/.env.development
```

### Required Variables

| Variable | Description | Example |
|---|---|---|
| `AUTH_HOST` | Auth service URL | `https://login.iblai.app` |
| `MENTOR_NEXTJS_HOST` | Mentor app URL (the app under test) | `https://mentorai.iblai.app` |
| `PLAYWRIGHT_USERNAME` | Test user email | `user@example.com` |
| `PLAYWRIGHT_PASSWORD` | Test user password | `your-password` |

### Optional Variables

| Variable | Description |
|---|---|
| `AUTH_FLOW` | Auth flow type: `username_password` (default), `magic_link`, `sso`, `direct_sso` |
| `AUTH_IDP` | Identity provider for SSO flows |
| `CANVAS_URL` | Canvas LMS URL (for canvas embed tests) |
| `CANVAS_EMAIL` / `CANVAS_PASSWORD` | Canvas credentials |
| `EMBED_URL` | URL to test embed widget injection |
| `ENABLE_STRIPE_TEST` | Set to `true` to run billing/subscription tests |
| `IBL_MENTOR_DISABLED_TABS` | Comma-separated list of disabled tabs |
| `PLATFORM` | Platform identifier for auth setup |

See [.env.example](.env.example) for the full list.

## Running Tests

```bash
# Run all tests
pnpm test:e2e

# Run with UI mode (interactive)
pnpm test:e2e:ui

# Run headed (see the browser)
pnpm test:e2e:headed

# Run a specific test file
pnpm test:e2e -- tests/chat/

# Run a specific project only
pnpm test:e2e -- --project=mentor-desktop-chromium

# Run with a specific grep pattern
pnpm test:e2e -- --grep "mentor card"
```

## Playwright Config

The config is generated by `createPlaywrightConfig` from `@iblai/iblai-js/playwright`. It creates projects for each browser (Chromium, Firefox, Safari) across two platforms:

| Platform | What it tests | Dependencies |
|---|---|---|
| `auth` | Auth flows (signup, login) | None |
| `mentor` | All mentor app tests | `setup` (auth) |

Extra projects:
- **`mentor-cleanup`** — Deletes test-created mentors after the mentor tests complete
- **`mentor-public-views`** — Tests mentor pages viewable by unauthenticated users

## Writing Tests

### Test File Location

Place test files in a feature directory under `e2e/tests/`:

```
e2e/tests/my-feature/
├── my-feature.spec.ts     # Test file (must end in .spec.ts)
└── helpers.ts             # Optional helpers specific to this feature
```

### Imports

```typescript
// Playwright test & expect
import { test, expect } from '@playwright/test';

// SDK utilities
import { logger, safeWaitForURL } from '@iblai/iblai-js/playwright';

// Shared env vars & helpers (from tests/utils.ts)
import { MENTOR_NEXTJS_HOST, authenticate, checkAdminStatus, waitForPageReady } from '../utils';

// Shared auth helpers (from tests/helpers.ts)
import { signUpWithEmailAndPassword, loginWithEmailAndPassword } from '../helpers';

// Reusable utility modules (from tests/utils/ directory)
import { fillCreateMentorForm } from '../utils/create-mentor';
import { navigateToLLMTab } from '../utils/navigate/navigate-to-llm-tab';
```

Note: `from '../utils'` resolves to `tests/utils.ts` (the main helpers file). `from '../utils/create-mentor'` resolves to `tests/utils/create-mentor.ts` (a sub-module).

### Test Template

```typescript
import { test, expect } from '@playwright/test';
import { MENTOR_NEXTJS_HOST, waitForPageReady } from '../utils';

test.describe('My Feature', () => {
  test.setTimeout(120_000);

  test.beforeEach(async ({ page }) => {
    await page.goto(MENTOR_NEXTJS_HOST);
    await waitForPageReady(page);
  });

  test('should do something', async ({ page }) => {
    // Test implementation
  });
});
```

### Available Helper Functions

From `tests/utils.ts`:

| Function | Description |
|---|---|
| `authenticate(page)` | Full login flow via auth SPA |
| `reAuthenticate(page, platformUrl)` | Re-login when tokens expire mid-test |
| `waitForPageReady(page)` | Wait for DOM complete + animations |
| `checkAdminStatus(page)` | Check if current user is admin via localStorage |
| `reliableClick(page, locator)` | Click with retry logic and stability waits |
| `reliableFill(page, locator, value)` | Fill input with retry and validation |
| `waitForDialogReady(page, locator)` | Wait for dialog to be fully loaded |
| `switchInstructorLearnerMode(page)` | Toggle instructor/learner mode switch |
| `expectNoAccessibilityViolations(page)` | Run axe-core accessibility scan |
| `searchMentor(page)` | Test mentor search flow |
| `mentorCardDisplay(page)` | Verify mentor cards render correctly |

### Tips

- Use `test.setTimeout()` for tests that involve auth redirects or AI responses (these can be slow)
- Use `waitForPageReady(page)` instead of `page.waitForLoadState('networkidle')` — SPAs keep making background requests that prevent networkidle from resolving
- Use `safeWaitForURL()` from the SDK for URL navigation waits — it handles both string patterns and regex
- Use `checkAdminStatus(page)` to conditionally run admin-only test steps
- Place feature-specific helpers in a `helpers.ts` file within the feature directory
- Auth state is saved per-browser in `playwright/.auth/` — tests in the `mentor` platform reuse this state
- Traces are automatically captured on failure (`retain-on-failure`) and uploaded to S3 with Trace Viewer links in the CI summary
